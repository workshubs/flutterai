# Flutter AI — منصّة توليد تطبيقات Flutter بالذكاء الاصطناعي

تطبيق ويب (Next.js + TypeScript) فيه:
- صفحة دخول متصلة بـ Supabase (بدون تسجيل حساب جديد — تنشئ المستخدمين يدويًا).
- صفحة بناء رئيسية: تكتب وصف التطبيق في صندوق الدردشة، فيظهر هيكل مشروع
  Flutter تدريجيًا على الجانب، وتُكتب محتويات `lib/main.dart` بتأثير
  "الكتابة الحية"، ثم تقدر تنزّل المشروع كملف zip.

## التشغيل عندك محليًا

1. ثبّت الحزم:
   ```
   npm install
   ```
2. انسخ ملف البيئة وعدّله:
   ```
   cp .env.local.example .env.local
   ```
   - بيانات Supabase موجودة مسبقًا فيه (رابط ومفتاح anon لمشروعك).
   - **لازم** تضع `AI_API_KEY` و`AI_API_URL` الخاصين بخدمة الذكاء الاصطناعي
     اللي تستخدمها، وإلا ستظهر الواجهة كود نموذجي مؤقت فقط (Fallback) حتى
     تجرّب التحريك والتصميم بدون أي اشتراك.
3. شغّل:
   ```
   npm run dev
   ```
4. افتح http://localhost:3000 — ستحوّلك تلقائيًا لصفحة الدخول.
5. أنشئ مستخدم تجريبي من لوحة Supabase: **Authentication → Users → Add user**.

## ⚠️ نقطة مهمة لازم تعرفها

شجرة الملفات (`android/`, `ios/`, `web/`, إلخ) المعروضة في الواجهة وملف الـ
zip اللي تنزّله هي **هيكل تمثيلي مبسّط فقط** — لأن ملفات Android (مشروع
Gradle كامل) وiOS (مشروع Xcode) الحقيقية تحتوي مئات الملفات، وبعضها ملفات
ثنائية (binary) يصدرها أمر `flutter create` نفسه باستخدام Flutter SDK —
وهذا غير متاح للتوليد اليدوي بدقة. كل مجلد منصة بداخل الـ zip حاليًا فيه
ملف `NOTE.txt` يوضح كيف تكمّله بأمر واحد على جهازك:
```
flutter create my_app
```
ثم تستبدل `lib/main.dart` بالنسخة المولَّدة من المنصّة.

**لحل هذا بشكل نهائي** (يجعل التنزيل مشروعًا حقيقيًا 100% جاهزًا للبناء):
شغّل `flutter create my_app` مرة واحدة بنفسك على جهاز فيه Flutter SDK،
وأرسل لي ذلك المشروع (أو فقط مجلدات android/ و ios/ منه)، وأنا أدمجه
كقالب ثابت يُستخدم تلقائيًا في كل تنزيل بدل ملاحظة NOTE.txt.

## ربط خدمة الذكاء الاصطناعي الفعلية

منطق الاستدعاء في `src/app/api/generate-dart/route.ts` مكتوب بصيغة
"OpenAI Chat Completions" الشائعة (`POST /v1/chat/completions` مع
`Authorization: Bearer <key>`). إذا خدمتك تتبع تنسيقًا مختلفًا، عدّل دالة
`callAiProvider` فقط — باقي الواجهة لن تتأثر.

## الأمان

- مفتاح Supabase الموجود (anon/public) مصمم ليكون عامًا، لا بأس من وجوده
  في الكود الأمامي، بشرط تفعيل قواعد RLS الصحيحة على جداولك.
- مفتاح الذكاء الاصطناعي (`AI_API_KEY`) **لا يوضع أبدًا** في كود الواجهة
  الأمامية — هو موجود فقط في `.env.local` ويُستخدم داخل مسار API على
  السيرفر، فلا يصل لمتصفح المستخدم نهائيًا.
